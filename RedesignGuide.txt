Architectural Specification: Redesigning System Planners for Modern Skeuomorphic Industrial EnvironmentsDesign System Foundation: Palette, Skeuomorphism, and HierarchyTraditional web applications prioritize flat visual patterns, minimalist white spaces, and structural simplicity to minimize cognitive strain during brief user sessions. However, system-oriented calculators and interactive planning tools require a high-density, system-intensive layout. When users manage complex production pipelines, flat designs can hide the logical flow and hierarchy of the system. A modern, skeuomorphic design system solves this by transforming abstract digital interfaces into tactile, structured control surfaces that reflect the weight, texture, and industrial layout of physical machinery.An effective skeuomorphic design system relies on a dark, high-contrast palette. The interface uses deep charcoal and warm slate hues to prevent eye strain during long planning sessions. Accent colors are applied selectively to draw attention to active components, operational bottlenecks, or specific resource streams. Rather than choosing arbitrary decorative accents, this design system uses color as a critical tool for displaying real-time data, indicating machine statuses, and categorizing resource tiers.Primary Surface and Text PaletteThe primary interface surfaces are constructed using deep, low-vibrancy gray tones that form a neutral backdrop, allowing key indicators to stand out clearly. Mathematical Throughput and Module ModelingTo calculate precise production rates, the node calculation engine model must account for machine crafting speeds, productivity modules, and quality multipliers. Throughput formulas are calculated using standard notation, ensuring units ($\mathrm{u}/\mathrm{s}$) are kept distinct from scalar quantities :$$\dot{m}_{\mathrm{output}} = \dot{m}_{\mathrm{base}} \cdot c_{\mathrm{speed}} \cdot \left( 1 + \sum \mu_{\mathrm{prod}} \right) \cdot \left( 1 + \sum \mu_{\mathrm{speed}} \right) \, \mathrm{u}/\mathrm{s}$$Where $\dot{m}_{\mathrm{output}}$ is the final calculated item throughput , $\dot{m}_{\mathrm{base}}$ is the base recipe output rate, $c_{\mathrm{speed}}$ is the core machine crafting multiplier, $\sum \mu_{\mathrm{prod}}$ is the sum of active productivity modules , and $\sum \mu_{\mathrm{speed}}$ represents active speed module effects.Re-Engineering Custom React Flow NodesCustom nodes are styled as compact control panels. They display active utilization rates using progress bars that feature a bright-green indicator sliding across a recessed, dark slot. Integrated module slots show active productivity or speed multipliers :Tactile Node Header: Features a solid subheader bar (subheader_frame) displaying the machine's primary function icon and its current efficiency percentage.Port Handles (Handles): Replaced with metallic circular sockets. These handles turn bright yellow on hover to indicate valid connection points.Conveyor Belt Edges: Custom React Flow edges are built as dual-layered SVG paths. A thick, dark-slate background path represents the physical belt, while an animated dashed layer on top represents item flow. The speed of the chevron animation scales dynamically based on calculated throughput. If a line bottlenecks or runs out of resources, the chevron animation stops, providing instant visual feedback on the state of the factory.Production-Ready Custom Edge ImplementationJavaScriptimport React, { useMemo } from 'react';
import { getBezierPath } from 'reactflow';

// Custom conveyor edge displaying dynamic speed animations based on throughput
export const ConveyorBeltEdge = ({
  id,
  sourceX,
  sourceY,
  targetX,
  targetY,
  sourcePosition,
  targetPosition,
  style = {},
  data = {},
}) => {
  const [edgePath] = getBezierPath({
    sourceX,
    sourceY,
    sourcePosition,
    targetX,
    targetY,
    targetPosition,
  });

  // Calculate dynamic flow rate duration
  // If throughput is zero, animation is disabled
  const movementDuration = useMemo(() => {
    const rate = data.throughput || 0;
    if (rate <= 0) return 0;
    // Maps the flow rate to animation speed: higher rates animate faster
    return Math.max(0.8, 12 - (rate / 5)); 
  }, [data.throughput]);

  return (
    <>
      {/* Background track: represents the physical conveyor belt */}
      <path
        id={`${id}-chassis`}
        d={edgePath}
        fill="none"
        stroke="#1a1a1a"
        strokeWidth={10}
        className="react-flow__edge-path"
      />
      
      {/* Midground color track: representing the material or fluid type */}
      <path
        id={`${id}-well`}
        d={edgePath}
        fill="none"
        stroke={data.resourceColor || '#5f668c'}
        strokeWidth={6}
        opacity={0.85}
      />

      {/* Foreground flow animation layer: represents moving materials */}
      {movementDuration > 0 && (
        <path
          id={`${id}-flow`}
          d={edgePath}
          fill="none"
          stroke="#ffffff"
          strokeWidth={4}
          strokeDasharray="6, 14"
          style={{
            animation: `conveyorFlow ${movementDuration}s linear infinite`,
           ...style,
          }}
          opacity={0.7}
        />
      )}
    </>
  );
};
